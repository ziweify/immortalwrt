# AI 问答记录规则

## 规则概述

每次用户提问时，AI 助手会自动在 `./项目说明` 目录下创建记录文件，详细记录问题、解答和相关资料。

## 目录结构

```
项目说明/
├── README.md                    # 目录说明和索引
├── AI规则说明.md                # 本文件
├── 20260110/                    # 日期文件夹（YYYYMMDD格式）
│   ├── 01-如何编译x86-64平台镜像.md
│   ├── 02-为什么项目中有个乱码文件夹.md
│   └── 03-删除乱码文件夹.md
├── 20260111/                    # 另一天的记录
│   └── 01-xxx.md
└── ...
```

## 文件创建规则

### 1. 日期文件夹命名

在 `./项目说明` 目录下，按日期创建文件夹：

**格式**: `YYYYMMDD`

**示例**:
- `20260110` - 2026年1月10日
- `20260111` - 2026年1月11日
- `20261225` - 2026年12月25日

### 2. 记录文件命名

在对应的日期文件夹中，创建记录文件：

**格式**: `当日提问索引-标题.md`

**组成部分**:
- **当日提问索引**: 从 `01` 开始递增，同一天的第一个问题为 `01`，第二个为 `02`，以此类推
- **标题**: 问题的简短描述（中文），便于识别
- **扩展名**: `.md` (Markdown 格式)

**示例**:
- `01-如何编译x86-64平台镜像.md` - 当天的第1个问题
- `02-配置网络相关问题.md` - 当天的第2个问题
- `03-添加新软件包.md` - 当天的第3个问题

### 3. 完整路径示例

```
项目说明/20260110/01-如何编译x86-64平台镜像.md
项目说明/20260110/02-为什么项目中有个乱码文件夹.md
项目说明/20260111/01-配置LuCI界面.md
```

## 文件内容要求

每个记录文件必须包含以下部分（按顺序）：

### 1. 用户问题
- 完整记录用户的原始问题

### 2. 项目信息
- 项目名称、类型、位置
- 相关背景信息

### 3. 问题分析
- 对问题的理解和分析
- 涉及的技术领域

### 4. 执行步骤
- 实际执行的命令
- 执行结果（成功/失败）
- 关键输出信息

### 5. 详细解答
- 完整的技术解答
- 分步骤的详细说明
- 代码示例和配置示例

### 6. 技术细节
- 相关的配置文件位置
- 关键配置参数
- 技术实现细节

### 7. 常见问题
- FAQ 部分
- 问题排查方法
- 注意事项

### 8. 相关文件
- 涉及的文件列表
- 创建的新文件

### 9. 参考资源
- 相关文档链接
- 官方资源
- 社区资源

### 10. 后续步骤
- 待完成的任务清单
- 下一步操作建议

## 执行流程

1. **检测提问**: AI 识别到新的用户提问
2. **获取日期**: 获取当前系统日期，格式化为 `YYYYMMDD`
3. **检查日期文件夹**: 检查 `项目说明/YYYYMMDD/` 是否存在，不存在则创建
4. **确定索引**: 检查当日文件夹中已有文件，确定新的索引号（从 `01` 开始递增）
5. **创建文件**: 按照命名规则 `索引-标题.md` 创建新文件
6. **记录内容**: 按照内容结构记录完整信息
7. **更新索引**: 更新 `README.md` 中的索引列表

## 目录结构优势

### 按日期组织
- ✅ 便于按时间查找
- ✅ 每天的问题集中在一个文件夹
- ✅ 避免根目录文件过多

### 简化文件名
- ✅ 文件名更简洁（不需要包含日期）
- ✅ 索引号清晰，便于排序
- ✅ 标题更突出

## 注意事项

1. **日期格式**: 严格使用 `YYYYMMDD` 格式，8位数字，无分隔符
2. **索引递增**: 同一天内索引从 `01` 开始递增，不跳过
3. **唯一性**: 确保文件名唯一，同一天内索引递增
4. **完整性**: 每个文件应包含完整的上下文信息
5. **可读性**: 使用清晰的 Markdown 格式
6. **可搜索**: 标题和内容应便于搜索
7. **及时更新**: 每次创建新文件后更新 README.md 索引

## 维护建议

- 定期检查文件命名是否规范
- 确保索引文件及时更新
- 可以按年份创建子目录（如果文件过多，但当前不需要）
- 定期备份重要记录

## 迁移说明

如果从旧格式迁移到新格式：
- 旧格式: `项目说明/2026-01-10-01-标题.md`
- 新格式: `项目说明/20260110/01-标题.md`

需要：
1. 提取日期部分，转换为 `YYYYMMDD` 格式
2. 提取索引和标题部分
3. 创建对应的日期文件夹
4. 移动并重命名文件
